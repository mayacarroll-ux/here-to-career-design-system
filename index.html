Here to Career Design System

Design tokens and components for the redesigned Here to Career web app. Colors come from the Figma JA Primitives palette: turquoise as the primary action color, blue-black for headings, and a warm accent set. Component class names mirror Bootstrap 4 so styles drop onto the existing markup unchanged.

Animated Icons

One illustrated, full-color Lottie icon per career cluster. They loop a subtle "breathing" pulse and bring the color — shown here on a dark surface, the color-hierarchy pattern in action. Played via the LottieFiles web component (the only JS in the system).

<script src="https://unpkg.com/@lottiefiles/lottie-player@2.0.8/dist/lottie-player.js"></script>

<lottie-player src="assets/icons/construction.json"
  background="transparent" speed="1" loop autoplay></lottie-player>

Static versions

Same artwork as flat WebP in assets/icons/static/ — a no-JS fallback for email, print, dense lists, or anywhere the player isn't loaded. Transparent background, so they sit on any surface.

<img src="assets/icons/static/construction.webp" alt="Construction" width="64" height="64">

Note: agriculture.json and agriculture.webp currently hold the Advanced Manufacturing artwork (flagged ⚠). Drop corrected exports at the same paths and they update everywhere automatically.

Getting started

Two plain CSS files, no build step, no framework dependency.

<link rel="stylesheet" href="tokens.css">  <!-- primitives + semantic tokens -->
<link rel="stylesheet" href="h2c.css">     <!-- components (needs tokens) -->
<link href="https://fonts.googleapis.com/css2?family=Montserrat:wght@400;500;600;700;800&display=swap" rel="stylesheet">

<body class="h2c"> <!-- scope base typography -->

Components read only semantic --h2c-* tokens, which alias the --ja-* primitives. To reskin, change the aliases in one place — never hardcode a hex.
